# 오버래핑 가설 및 독립 판독 경로

Phase A에서 반드시 다루는 검증이다. Phase B에서는 `B-INT5`, `B-R5` 등으로 회귀한다.

## 배경

[05_qr-layering-feasibility-protocol.md](../../brainstorming/05_qr-layering-feasibility-protocol.md)의 Novelty Hypotheses 중 다음이 QR 오버래핑의 기술적 차별점이다.

1. 시각적 동일성을 유지하면서 다층 데이터 해석 경로를 분리할 수 있다.
2. 레이어별 접근 정책을 적용할 수 있다.
3. 사용 환경에서 허용 가능한 인식 성능을 유지할 수 있다.

## 검증 항목 (Phase A)

### O-1: 레이어 분리 (A-HYP1)

- 단일 심볼(또는 동일 시각 출력)에서 A/B/C가 논리적으로 분리되는지 정의한다.
- 인코딩 포맷(직렬화 스키마)과 디코딩 단계가 문서화되어야 한다.

### O-2: 일반 vs 확장 판독 (A-HYP2)

- **일반 경로**: 휴대폰 기본 카메라·상용 QR 앱 등 “사용자가 추가 소프트웨어 없이” 기대하는 경로에서 Layer A만 노출되는지.
- **확장 경로**: PoC 디코더 또는 지정 앱에서 B/C 복원이 가능한지.
- 실패 기준: 일반 경로에서 B/C 원문이 노출되면 정책 위반으로 기록한다.

### O-3: Baseline 대비 성능 (A-HYP3)

- 동일 ECC·근사 payload 길이의 단일 레이어 QR과 비교해 스캔 성공률·레이턴시 저하가 허용 범위인지 측정한다.
- 허용 범위는 `metrics-protocol.md`의 임계값과 정렬한다.

### O-4: 시각적 동일성 (A-HYP4)

- “동일 QR 이미지”의 정의를 고정한다(픽셀 동일 vs 인쇄 허용 오차 vs 동일 버전/모듈 등).
- 오버래핑이 시각적 합성·오버레이를 쓰는 경우, baseline과 비교할 지표를 명시한다.

## 독립 판독 경로 (Phase A 스파이크 + Phase B B-INT5)

**목적**: `encode → 동일 구현체 decode` 라운드트립만으로는 외부 세계와의 호환을 증명하지 못한다.

| 단계 | 검증 |
|------|------|
| A | 생성 파이프라인 출력을 최소 1개의 **독립** 판독기(다른 라이브러리, 다른 언어 런타임, 또는 기본 카메라 앱)로 읽어 Layer A 일치 확인 |
| B | `B-INT5`: Phase A에서 고정한 독립 경로를 CI 또는 수동 게이트로 반복 |

## Phase B 회귀 매핑

- `B-INT5`: 독립 경로 호환성 통합 테스트
- `B-R5`: 위조·부분 유효·만료 컨텍스트에서 상위 레이어 비노출
