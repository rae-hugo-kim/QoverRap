# AC 4지표 측정 프로토콜 (고정)

**상태**: Phase A에서 승인하고, Phase B `B-EXP1`~`B-EXP5`의 유일한 수치 기준으로 사용한다.

**출처**: [kickoff-summary.md](../harness/kickoff-summary.md), [05_qr-layering-feasibility-protocol.md](../../brainstorming/05_qr-layering-feasibility-protocol.md)

## 공통

- **환경 메타데이터** (모든 실험 run에 필수): Python/OS, `qrcode`·`pyzbar`(또는 사용 라이브러리) 버전, QR ECC·버전, payload 바이트 길이(레이어별), 커밋 해시 또는 보고서 ID.
- **재현성**: 동일 시드가 있으면 기록하고, 무작위 변조 실험은 시드 고정을 권장한다.

## MET-1: 스캔 성공률 (AC: 95%+)

| 항목 | 고정 값 |
|------|---------|
| 목표 | 성공 스캔 수 / 시도 수 >= **0.95** |
| 시도 수 | **200회** (AC 문구와 일치) |
| 디바이스 | 최소 **2종** 이상 다른 기기(예: 서로 다른 OS 또는 카메라 모듈). 동일 기기만으로는 AC 문구 “다양한 디바이스”와 불일치 가능 — 보고서에 한계를 명시한다. |
| 조명 | 최소 **2조건** (예: 실내 일반 / 저조도 / 역광 중 택2). 조건별 시도 횟수를 나눠 기록한다. |
| 성공 정의 | 지정 디코더(또는 기본 카메라 앱)가 **Layer A** payload를 오류 없이 읽음 |
| 실패 정의 | 타임아웃·무응답·체크섬 실패·완전 미스캔 |

## MET-2: 디코드 레이턴시 (AC: 300ms 이하)

| 항목 | 고정 값 |
|------|---------|
| 목표 | **p95** 디코드 시간 <= **300 ms** (AC의 “≤300ms”를 분위수로 고정; 평균만으로는 통과 판정하지 않음) |
| 측정 구간 | 이미지 바이트가 메모리에 로드된 시점부터 **순수 디코드+검증(parse+verify)** 완료까지. 파일 I/O는 별도 필드로 기록하되 AC 판정에서 제외한다. |
| 워밍업 | 각 프로세스당 최소 1회 워밍업 후 측정 |
| 보고 | 평균, p50, **p95**, p99, 시행 횟수 |

## MET-3: 레이어 무결성 (AC: 99%)

| 항목 | 고정 값 |
|------|---------|
| 목표 | 변조 샘플에 대해 **탐지(거부 또는 실패 플래그)** 비율 >= **0.99** |
| 표본 | 최소 **200건** 변조 시도 권장 (99% 신뢰를 문서적으로 뒷받침하기 위함; 더 작으면 보고서에 불확실성 명시) |
| 변조 유형 | 최소 3종: payload 비트 플립, 서명 제거, 다른 키로 재서명 시도(가능 시) |
| 정상 샘플 | 오탐 측정을 위해 정상 샘플 **50건** 이상에서 “허용”이어야 함 |

## MET-4: 컨텍스트 라우팅 정확도 (AC: 98%+)

| 항목 | 고정 값 |
|------|---------|
| 목표 | 올바른 레이어 집합 반환 비율 >= **0.98** |
| 케이스 수 | **100개** 조건 분기 (AC·05 문서와 일치) |
| 생성 규칙 | public / authenticated / verified / unknown을 **명시적 비율 또는 격자**로 생성하고 문서에 붙인다. |
| 판정 | 각 케이스에서 기대 집합 {A}, {A,B}, {A,B,C}와 실제 집합이 일치하면 성공 |
