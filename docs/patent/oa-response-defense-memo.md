# OA 응답 방어 논거 메모 (출원 후 거절이유통지 대비)

> **목적**: 본인 출원 (KEAPS) 직후 KIPO 심사관이 청구항 1 단독의 진보성에 대해 거절이유통지 (OA) 를 발부할 경우의 답변 논거 및 보정안 후보. 본 memo 는 출원 문서에는 미반영. 출원 후 60일 의견서 제출 기한 내에 본인이 활용.
>
> **상태**: 16차 라운드 (2026-05-15) Codex 권고 채택, 본 memo 신설.

---

## 1. 청구항 1 단독 진보성 거절 시나리오

청구항 1 (broad 폭, 생성 방법) 은 14·15·16차 라운드를 거쳐 wire format 구조한정이 강화되었으나, safe-fallback 결합 functional language 가 제거된 순수 생성 청구항이다. 한국 심사관이 다음 조합으로 진보성 거절을 시도할 가능성이 있다.

| 조합 | 인용 선행기술 | 공격 논리 |
|------|------------|----------|
| A | SAP signed QR (비특허문헌 0007) + base64 delimiter framing | "selected QR data + JWS 서명 + delimiter trailer 결합은 공지 기술의 단순 조합" |
| B | SMART Health Cards (비특허문헌 0004) + 일반 binary header 직렬화 | "전체 credential JWS 서명 + 헤더 첨부는 공지" |
| C | ICAO VDS (비특허문헌 0005) + delimiter/base64 | "VDS 의 signed payload + 일반 인코딩 직렬화 조합" |
| D | SQRC (비특허문헌 0003) + (가)군 다층 QR (비특허문헌 0001, 0002) | "다층 QR 코드 + 비공개 영역 결합" |

각 조합 모두 청1 의 wire format 결합이 "통상적 직렬화" 라는 공격 가능.

## 2. 의견서 1차 답변 논거 (broad 청1 유지)

**핵심 주장**: 청1 의 진보성은 "각 구성요소의 공지 여부" 가 아니라 "**유기적 결합**" 에 있다. 특허법 §29(2) + 대법원 판례 (2014후2061 계열) 는 진보성 판단 시 구성요소를 분해해 공지 여부만 보지 말고 결합의 곤란성·효과를 보아야 함을 판시.

본 발명의 결정적 결합:
1. **선두 평문 호환성**: Layer A 가 표준 QR 리더 판독 결과의 *선두* 에 평문으로 위치 → (마)군 (SHC, VDS, SAP) 의 전체 메시지 단일 서명 객체와 본질적으로 다름. (마)군은 판독 결과 전체가 의미 없는 base64/JWS 문자열로 노출되어 표준 리더 사용자가 즉시 이해 불가.
2. **versioned binary header 의 wire format 내부 경계 결정**: 후속 트레일러 디코딩 없이도 Layer A 분리 가능 + 디코딩 후 B/C 바이트 경계 결정 가능. (마)군은 통상 단일 서명 객체로 trailer 구획이 없거나 메시지 전체를 base64 로 감싸므로 본 결합 효과 없음.
3. **공유 직렬화 + 분리 가능성의 동시 충족**: 표준 QR 리더 호환 + 본 발명 디코더 호환의 양립이 (가)~(다)군의 모듈/색상 변조 방식과 (마)군의 전체 서명 방식 어디에도 개시되지 아니함.

**효과 주장**: 본 결합으로 "표준 리더 사용자도 Layer A 의미 인지 가능 + 본 발명 디코더 사용자는 B/C 검증 결과 활용 가능" 의 두 사용자 군이 *동일 페이로드 문자열로 동시 서비스* 가능. 종전 기술 어디에도 이 이중 호환성이 개시되지 않음.

## 3. Fallback 보정 청구항 1' 후보 (broad 청1 거절 확정 시)

심사관이 위 1차 답변을 수용하지 않을 경우 보정으로 좁힐 수 있는 fallback 독립항 후보. 14차 Codex 권고 + 16차 Codex 추가 권고 종합.

### 3.1 청구항 1' (canonical signing 결합)

> **[청구항 1']** 1 이상의 프로세서 및 1 이상의 메모리를 포함하는 컴퓨팅시스템에 의하여 수행되는, QR 코드의 표준 심벌 체계에 의해 인코딩 가능한 단일 페이로드 문자열을 생성하는 방법으로서,
>
> 표준 QR 코드 데이터로 인코딩되었을 때 표준 QR 리더의 판독 결과 문자열의 선두에 평문으로 위치하는 공개 텍스트 계층인 Layer A를 입력받는 단계;
>
> Layer A와 구별되는 컨텍스트 메타데이터 계층인 Layer B를 입력받거나, 또는 Layer B를 입력받지 아니하는 단계 — 양자 포함 모드 / 양자 부재 모드;
>
> 상기 양자 포함 모드에서, **미리 정해진 매직 식별자, 1 바이트 버전 필드, Layer A의 UTF-8 바이트 길이를 나타내는 길이 필드, Layer A의 UTF-8 바이트열, Layer B의 바이트 길이를 나타내는 길이 필드, 및 Layer B의 바이트열을 결정론적인 순서로 결합한 정규화된 서명 대상 메시지에 대하여** 사전 보유된 비밀키로 서명함으로써 디지털 서명 계층인 Layer C를 생성하는 단계;
>
> 상기 양자 포함 모드에서, **1 바이트 버전 필드, 2 바이트 Layer B 길이 필드 및 2 바이트 Layer C 길이 필드를 빅엔디안 부호없는 정수로 직렬화한 5 바이트 이진 헤더** 를 생성하는 단계;
>
> 상기 양자 포함 모드에서, 상기 이진 헤더, Layer B, 및 Layer C 를 순차적으로 결합한 프레임을 **base64 인코딩** 방식으로 인코딩하여 트레일러를 생성하는 단계;
>
> 상기 양자 포함 모드에서, Layer A, 미리 정해진 구획자, 및 상기 트레일러를 순차적으로 결합하여 단일 페이로드 문자열을 출력하는 단계; 및
>
> 상기 양자 부재 모드에서, 구획자 및 트레일러를 생성하지 아니하고 Layer A 만으로 단일 페이로드 문자열을 구성하여 출력하는 단계;를 포함하고,
>
> 상기 단일 페이로드 문자열은, Layer A 가 상기 구획자 이전에 배치되어 트레일러 디코딩 없이도 분리 가능하고, 상기 이진 헤더의 길이 필드가 Layer B 및 Layer C 의 바이트 경계를 결정 가능한 wire format 구조를 갖는 것을 특징으로 하는, 단일 QR 페이로드 생성 방법.

**좁힘 효과**:
- 청12 의 canonical signing message (매직·버전·길이·바이트열 결정론적 결합) 결합 → (마)군의 통상 JWS/COSE compact serialization 과 명시적 차별
- 5바이트 헤더 (1+2+2) 의 구체적 비트 직렬화 → 통상 직렬화로의 reduce 불가
- base64 인코딩 명시 → 임의 텍스트 안전 인코딩의 추상 한정 회피

**잃는 폭**: alternative signing algorithm 구체화 (Ed25519 외), 임의 header layout, 비-base64 텍스트 안전 인코딩 — 다만 본 명세서 §8.1 본문이 이 wire format 만 구체 기재하므로 좁힘 후 청구범위 폭이 본문 기재 범위에 정확히 일치.

### 3.2 청구항 1'' (minimal narrowing — 5바이트 헤더 + base64 만 결합)

청1' 가 과도하게 좁다고 판단되면 더 가벼운 보정:

> **[청구항 1'']** ... [청1 본문 유지] ...
>
> 단, 상기 이진 헤더는 1 바이트 버전 필드, 2 바이트 Layer B 길이 필드 및 2 바이트 Layer C 길이 필드를 빅엔디안 부호없는 정수로 직렬화한 5 바이트로 구성되고, 상기 텍스트 안전 인코딩 방식은 base64 인코딩인 것을 특징으로 하는, 단일 QR 페이로드 생성 방법.

본 형식은 현재 청3 종속항을 청1 본문에 통합한 것에 해당. 청3 의 진보성 근거 (§8.1 본문) 와 동일하게 뒷받침됨.

## 4. 의견서 답변 시 인용할 본 명세서 위치

| 논거 | 본 명세서 위치 (application) |
|------|-------------------|
| 선두 평문 호환 효과 | §0031 효과 + §0044 본문 |
| versioned binary header 의 결정성 | §0045 본문 (1+2+2 바이트 구조) + 청3 |
| canonical signing message 의 결정적 결합 | §0049~§0052 본문 (§8.1.1 정규화) + 청12 |
| (마)군과의 차별 — 전체 메시지 단일 서명 객체 대비 | §0011, §0006 배경기술 + 비특허 0003~0007 |
| 두 사용자 군 (표준 QR 리더 / 본 발명 디코더) 동시 호환 | §0023 ~ §0024 본문 |

## 5. 보정 시기 전략

1. **1차 의견서**: broad 청1 유지 + §2 의견서 답변 논거 제출. 보정안 미제출.
2. **2차 의견서 (1차 불수용 시)**: 청1' (canonical signing 결합) 보정안 제출.
3. **거절 결정 후 재심사 청구 시**: 청1'' (minimal narrowing) 또는 분할출원 (broad 청1 폐기 + 좁은 종속항 chain 단독 청구) 검토.

## 6. 본 memo 활용 시 주의

- 본 memo 의 보정안 청1'/청1'' 는 본 명세서 §8.1 + §8.1.1 본문에 모두 뒷받침되므로 §47(2) 신규사항 위반 위험 없음.
- 단, 보정 시점에 청1 의 broad 폭을 포기하면 침해 분석 시 권리범위가 좁아진다. 보정은 거절이유 회피 최후 수단으로 사용.
- 청1 유지 + 종속항 (청7·8·12·13) 으로 방어하는 portfolio 전략이 기본 — 청1 거절이라도 청2 (독립 검증항) 및 종속항들로 등록 가능.
