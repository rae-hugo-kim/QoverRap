# KIPO 직접 출원 (KEAPS) 가이드

본 문서는 KIPO 특허청에 본인 명의로 직접 출원할 때 KEAPS (특허청 전자출원 소프트웨어) 에 본 명세서를 입력하는 절차를 설명한다. 변리사 위임 출원이 아닌 **본인 출원** 시나리오를 가정한다.

> **법률 자문 아님**. 본 가이드는 KEAPS 사용 절차 안내이며 출원 전략·청구항 표현·법률 적합성에 대한 자문을 포함하지 않는다.

---

## 1. 사전 준비

### 1.1 본인 인증 수단

KEAPS 로그인 및 전자서명을 위해 다음 중 하나가 필요하다.

- **공동인증서 (구 공인인증서)**: 본인 명의 발급 (은행 / 인증기관 발급)
- **간편인증**: 카카오·PASS·네이버 등 (KEAPS 일부 절차에 한해 가능)
- **GPKI**: 공무원 / 공공기관용 (해당 없으면 무시)

### 1.2 출원인 정보 사전 확보

| 항목 | 자료원 | 비고 |
|------|--------|------|
| 출원인 코드 | KEAPS 첫 회 출원 시 자동 발급 | 이후 모든 출원에 동일 코드 사용 |
| 출원인 성명 | 주민등록 등 본인 명의 | |
| 출원인 주소 | 우편물 수령 가능 주소 | |
| 출원인 주민번호 (개인) 또는 사업자번호 (법인) | | KEAPS 가 검증 |
| 발명자 정보 | 출원인과 동일하면 동일 입력. 다르면 별도 항목 | 본 발명은 출원인 = 발명자 가정 |

### 1.3 KEAPS 다운로드 및 설치

- 공식: **`https://www.kipo.go.kr/`** → 전자출원 → KEAPS 다운로드
- Windows 전용 (Mac / Linux 미지원). WSL/VM 등 Windows 환경 준비 필요
- 설치 후 첫 실행 시 본인 인증서 등록

## 2. 본 명세서 → KEAPS 입력 매핑

KEAPS 의 각 입력 화면과 본 저장소 (`docs/patent/spec-draft.md`) 의 섹션 대응표.

| KEAPS 입력 항목 | 본 명세서 섹션 | 비고 |
|----------------|--------------|------|
| **(서지)** 발명의 명칭 | §1 발명의 명칭 (한국어 + 영문) | 한·영 분리 입력 |
| **(서지)** 출원인 정보 | (KEAPS 자체 입력) | §1 외 명세서 외부 |
| **(서지)** 발명자 정보 | (KEAPS 자체 입력) | |
| 명세서 — 기술분야 | §2 | 그대로 복사 |
| 명세서 — 배경기술 | §3 (3.1 ~ 3.3) | 5군 비교표는 표 그대로 입력 가능 |
| 명세서 — 해결하려는 과제 | §4 | 4 항목 그대로 |
| 명세서 — 과제의 해결수단 | §5 (a-d) | 그대로 |
| 명세서 — 발명의 효과 | §6 | 표 그대로 |
| 명세서 — 도면의 간단한 설명 | §7 (Fig. 1 ~ Fig. 8) + §7.1 참조부호 일람표 | 참조부호 표는 KEAPS 의 "도면의 간단한 설명" 끝에 포함 |
| 명세서 — 발명을 실시하기 위한 구체적 내용 | §8 (8.1 ~ 8.6) | ASCII 다이어그램은 텍스트 그대로 입력 가능 (KEAPS 가 모노스페이스 폰트로 보존) |
| 청구범위 | §9 (청구항 1 ~ 11) | 청구항 번호별 KEAPS 입력 필드에 1:1 매핑 |
| 요약서 | §11 Abstract | 약 400자 이내. 본 §11 이 적정 길이 |
| 도면 | `docs/patent/figures/rendered/*.png` | Fig. 1, 2, 3, 4, 8 + (선택) Fig. 5~7 |

> **§3.2 5군 분류표**, **§9 청구항 표**, **§10 신규성 매핑표**: KEAPS 는 표 입력을 지원하지만, 일부 환경에서 표가 깨질 수 있다. 표가 깨지면 평문 항목 나열로 변환하여 입력.

## 3. 청구범위 입력 형식

KEAPS 청구범위 입력은 청구항 번호별로 분리된 필드에 입력한다. 본 명세서 §9 의 청구항 1 ~ 11 을 그대로 옮긴다.

### 3.1 청구항 인용 형식 표준화

KEAPS / KIPO 출원 표준 인용 형식:

| 본 명세서 표기 | KEAPS 표기 |
|---------------|-----------|
| 청구항 1 에 있어서, | 제 1 항에 있어서, |
| 청구항 2 의 | 제 2 항의 |
| 청구항 1 또는 청구항 2 에 있어서, | 제 1 항 또는 제 2 항에 있어서, |

**일괄 변환**: KEAPS 입력 직전에 다음 sed/find-replace 일괄 적용:
- `청구항 1` → `제 1 항`
- `청구항 2` → `제 2 항`
- `... 청구항 11` → `제 11 항`

### 3.2 청구항 본문 들여쓰기

본 명세서의 청구항은 (a)/(b)/(c) 의 단계 구분을 사용한다. KEAPS 는 평문이므로, 단계 구분 들여쓰기는 줄바꿈 + 두 칸 공백으로 유지하면 된다.

### 4.3 도면 KEAPS 첨부 시 형식

- **파일 형식**: PNG (KEAPS 지원). PDF, JPEG 도 가능하나 PNG 권장
- **해상도**: 300dpi 이상 (본 PNG 모두 충족)
- **파일명**: KEAPS 자체 명명 규칙 (자동 부여), 본 저장소 파일명은 참고용
- **도면 번호**: KEAPS 입력 시 "도 1", "도 2", ... 순서로 부여 (본 Fig. 1 = "도 1", ...)

## 7. 제출 후 절차 개요

1. **출원번호 부여**: 출원 직후 자동 부여 (예: 10-2026-XXXXXX)
2. **방식심사**: 1~2주 — 형식 결함 통지 가능 (도면 누락, 청구범위 형식 오류 등)
3. **공개**: 출원일로부터 1년 6개월 후 자동 공개 (또는 조기공개 신청)
4. **실체심사 청구**: 출원일로부터 3년 이내 신청 가능. 신청해야 심사 시작
5. **심사관 의견서**: 통상 6~18개월 후. **거절이유통지** 시 의견서 제출 (60일 이내)
6. **등록 결정 또는 거절 결정**: 심사 결과
7. **등록료 납부**: 등록 결정 시 3년분 등록료 (개인 약 60,000원선)

## 8. 출원 직전 최종 체크리스트

```
[ ] §1 발명의 명칭 — 한국어 + 영문 동일 의미 확인
[ ] §2 ~ §6 — 평문 일관성 (마지막 검토)
[ ] §7 ~ §7.1 — 도면 설명과 참조부호가 실제 도면 (rendered/*.png) 과 일치
[ ] §8.1 ~ §8.6 — ASCII 다이어그램이 KEAPS 모노스페이스 표시에서 깨지지 않는지
                  (테스트 환경에서 확인 권장)
[ ] §9 청구항 — "청구항 N" → "제 N 항" 일괄 변환 적용
[ ] §10 신규성 매핑 — 표 깨짐 시 평문 변환
[ ] §11 요약서 — 글자 수 (400자 이내)
[ ] 도면 PNG 5개 (필수) — Fig. 1, 2, 3, 4, 8
[ ] 도면 PNG 6개 (선택) — Fig. 5, 6a-c, 7a-b — 추가 수동 정리 후 첨부 여부 결정
[ ] 출원인 정보, 발명자 정보 KEAPS 등록
[ ] 본인 인증서 / 간편인증 준비
[ ] 출원료 납부 수단 (계좌 또는 카드) 준비
```
